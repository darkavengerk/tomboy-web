# 체크박스(체크리스트) 기능 설계

- 날짜: 2026-05-20
- 상태: 설계 승인됨, 구현 계획 대기

## 1. 목적

노트 안에서 할 일 목록을 체크박스로 관리한다. `체크리스트:` 로 시작하는
헤더 줄 다음에 오는 리스트 항목들이 불릿 대신 클릭 가능한 체크박스로
표시되고, 클릭하면 완료/미완료가 토글된다. `Ctrl+P` 단축키로 체크리스트
블록을 빠르게 삽입할 수 있다.

기존 `TODO`/`Done` 영역 기능(`lib/editor/todoRegion/`)과는 **완전히 별개의
독립 기능**이다. 서로 감지 키워드가 다르고 동작도 겹치지 않는다.
체크박스 기능은 항목을 이동시키지 않고 제자리에서 상태만 토글한다.

## 2. 동작 모델

### 2.1 체크리스트 영역 감지

- 노트의 최상위 문단(제목 줄 제외) 중 `trim()` 한 텍스트가 `체크리스트:`
  로 **시작**하는 문단이 헤더다. 콜론은 필수이며, 콜론 뒤에 설명 텍스트가
  올 수 있다 (예: `체크리스트: 장보기`, `체크리스트: 2026 1분기 목표`).
- 헤더 문단 **바로 다음**에 오는 1개 이상의 연속된 리스트 블록
  (`bulletList` / `orderedList`)이 그 영역의 리스트가 된다.
- 연속된 리스트 여러 개는 논리적으로 하나의 영역으로 묶는다.
- 헤더 다음에 리스트가 없으면 영역이 아니다 (그냥 일반 문단).
- 감지 구조는 기존 `todoRegion/regions.ts` 의 `findTodoRegions` 를
  그대로 미러링한다. 차이는 감지 정규식뿐: `/^체크리스트:/`.

### 2.2 체크리스트 항목

- 영역의 리스트 안에 있는 모든 `listItem` 이 체크리스트 항목이다.
  **중첩 항목 포함** — 깊이 제한 없이 재귀적으로 모두 체크박스가 된다.
  (todoRegion 은 깊이 1/2 만 다루지만, 체크박스는 단순 토글이라 깊이
  제한이 불필요하다.)
- 각 항목의 첫 문단 텍스트는 맨 앞에 4글자 마커를 가진다.

## 3. 저장 형식 (`.note` XML)

체크 상태는 항목 첫 문단 본문 맨 앞의 **4글자 텍스트 마커**로 저장한다.

| 상태 | 마커 |
|------|------|
| 미체크 | `[ ] ` (대괄호, 공백, 대괄호, 공백) |
| 체크   | `[X] ` (대괄호, 대문자 X, 대괄호, 공백) |

- 표준 `<list-item>` 안의 일반 텍스트일 뿐이다. `noteContentArchiver.ts`
  (XML ↔ TipTap JSON 변환) 는 **수정하지 않는다.** 라운드트립 안전.
- 마커 파싱 정규식: `/^\[([ xX])\] /`. 캡처가 `x` 또는 `X` 면 체크 상태.
  읽을 때는 소문자 `[x]` 도 체크로 인정하고, 쓸 때는 항상 `[X]` 로
  정규화한다.
- 마커는 라이브 ProseMirror 문서 안에 실제 텍스트로 존재한다. 따라서
  `editor.getJSON()`, autoLink 스캔, 복사/붙여넣기 직렬화 등 모든 경로가
  마커를 텍스트로 본다. 복사 시 평문에 `[X] 빨래 돌리기` 처럼 마커가
  포함되는데, 이는 의미 있는 표현이므로 그대로 둔다.

## 4. 화면 표시

선택된 방식: **깔끔한 체크박스 위젯 (마커 텍스트 숨김)**.

- 체크리스트 영역의 `listItem` 은 리스트 불릿을 숨긴다 (`list-style: none`).
  기존 todoRegion 처럼 `Decoration.node(liPos, liEnd, { class: ... })` 로
  `tomboy-checkbox-item` 클래스를 부여하고 CSS 에서 처리한다.
- 마커 4글자 범위에 인라인 데코레이션을 씌워 화면에서 숨긴다
  (`display: none`).
- 항목 첫 문단 시작 위치에 위젯 데코레이션으로 체크박스 컨트롤(`<button>`)
  을 그린다. 체크 상태에 따라 ☐/☑ 모양. `contenteditable="false"`,
  클릭 시 토글. 위젯 빌더는 todoRegion 의 `buildButton` 패턴을 미러링.
- 체크된 항목은 글자색을 살짝 흐리게 표시한다 (`opacity: 0.6` 수준).
  **취소선은 적용하지 않는다.** — 요청대로 "그냥 체크박스만".
- `체크리스트:` 헤더 문단 자체는 일반 텍스트로 둔다 (별도 스타일 없음).

## 5. 커서 & 마커 보호

마커가 라이브 문서 안의 실제 텍스트이고 화면에서는 숨겨지므로, 커서가
숨겨진 마커 안에 갇히지 않게 해야 한다. 마커 범위를
`(paraStart, paraStart+4)` 라 하자.

규칙:

1. **클릭** — `display: none` 텍스트는 캐럿 레이아웃 박스가 없으므로,
   줄을 클릭하면 브라우저가 자연히 마커 직후(`paraStart+4`)에 캐럿을
   놓는다. 추가 처리 불필요.
2. **선택 정규화** — 안전장치로, 트랜잭션 결과 커서가 마커 범위 안에
   엄격히 들어가면 플러그인의 `appendTransaction` 이 마커 뒤끝
   (`paraStart+4`)으로 스냅한다. (`titleUniqueGuard` 의 커서 스냅 패턴과
   동류.)
3. **Backspace** — 사용자가 인지하는 "줄 시작"은 `paraStart+4` 다. 거기서
   Backspace 는 마커 글자 삭제가 아니라 일반 리스트 항목 시작에서의
   동작(이전 항목과 병합 / 들여쓰기 해제)을 해야 한다. `TomboyEditor`
   의 기존 `handleKeyDown` 에서 처리한다.
4. **ArrowLeft / Home** — `paraStart+4` 에서 왼쪽으로 가면 마커를 건너뛰어
   이전 항목/이전 블록으로 이동한다. Home 은 `paraStart+4` 로 간다.
5. **마커 자동 복원** — 사용자가 어떻게든 마커를 일부/전부 지우면 6절의
   자동 부착이 `[ ] ` 를 다시 채운다.

> **구현상 위험 지점.** 이 절은 기능에서 가장 까다로운 부분이다. 구현
> 계획은 이 동작을 먼저 프로토타이핑·테스트해야 한다. `display: none`
> 텍스트와 PM 커서의 상호작용이 브라우저별로 예상과 다르면, 대비책으로
> 9절의 "대안 — 아카이버 경계 마커 합성"으로 전환한다.

## 6. 자동 부착 ("자동으로 붙는")

- 플러그인의 `appendTransaction` (autoWeekday 플러그인과 같은 방식)이
  체크리스트 영역을 스캔한다. 영역 안에서 첫 문단 텍스트가 유효한 마커로
  시작하지 않는 `listItem` 에 `[ ] ` 를 자동으로 앞에 붙인다.
- 이로써 다음이 자동 처리된다:
  - 영역 안에서 Enter 로 새 항목 생성 → 새 빈 항목에 `[ ] ` 부착.
  - 항목 붙여넣기.
  - `체크리스트:` 아래에 사용자가 직접 타이핑한 기존 항목 / 기존 노트를
    열었을 때.
- 마커가 깨졌거나 사라진 항목도 다음 트랜잭션에서 복원된다.

## 7. 단축키 `Ctrl+P`

- 커서가 있는 최상위 블록 다음에 `체크리스트:` 문단 + 항목 하나짜리
  `bulletList` 를 삽입하고, 커서를 그 항목 안(마커 뒤)에 둔다.
- 커서가 빈 비제목 문단에 있으면 그 문단을 대체한다 — 기존
  `insertTodoBlock`(Ctrl+O)의 "빈 줄을 블록으로 바꾸기" 동작을 미러링.
- 삽입되는 항목은 `[ ] ` 마커를 포함한 채로 생성하고 커서는
  `paraStart+4` 에 둔다.
- 브라우저 기본 인쇄 단축키(Ctrl/Cmd+P)는 `event.preventDefault()` 로
  막는다 — `TomboyEditor` 의 `handleKeyDown` Ctrl/Cmd 스위치에
  `case "p"` 추가.

## 8. 파일 구성

기존 `todoRegion/` 과 `autoWeekday/` 구조를 미러링한다.

신규:

- `app/src/lib/editor/checklist/regions.ts` — `findChecklistRegions(doc)`,
  `findChecklistItems(regions)`, 마커 파싱 헬퍼 (`parseMarker`). 순수 함수,
  단위 테스트 대상.
- `app/src/lib/editor/checklist/commands.ts` — `toggleCheckboxAt(editor,
  liPos)` (`[ ] `↔`[X] ` 교체), `insertChecklistBlock(editor)` (Ctrl+P).
- `app/src/lib/editor/checklist/plugin.ts` — ProseMirror 플러그인:
  `appendTransaction` (자동 부착 + 선택 정규화), 데코레이션 빌드
  (불릿 숨김 노드 데코, 마커 숨김 인라인 데코, 체크박스 위젯 데코).
- `app/src/lib/editor/checklist/index.ts` — `TomboyChecklist` Extension
  + 재export.

## 9. 대안 — 아카이버 경계 마커 합성 (채택 안 함, 대비책)

마커를 라이브 PM 문서가 아니라 **`listItem` 의 schema 속성**(`checked`)으로
들고, `noteContentArchiver.ts` 가 직렬화 시 `[ ] `/`[X] ` 를 붙이고
역직렬화 시 떼어내는 방식. 장점은 라이브 문서가 깨끗해 커서 처리가
불필요하다는 것. 단점은 아카이버에 영역 인지 로직이 들어가 현재의 순수
구조적 변환기라는 성격이 흐려지고, 직렬화/역직렬화 양쪽에 영역 감지기가
이중으로 생겨 라운드트립 버그 표면이 넓어진다는 것.

CLAUDE.md 가 아카이버의 무손실 라운드트립을 강하게 강조하므로 채택하지
않는다. 단, 5절의 커서 처리가 구현 중 너무 불안정하면 이 방식으로
전환한다.

## 10. 범위 밖 (YAGNI)

- 체크 시 항목 이동·정렬 (그건 기존 `TODO`/`Done` 영역 기능의 역할).
- 별도의 "완료" 영역, 진행률(n/m) 표시.
- `.note` XML 에 커스텀 태그/속성 추가.
- 완료 항목 취소선.
- 체크박스 영역 외(일반 리스트)에서의 체크박스.

## 11. 불변식 / 주의

- 기존 `todoRegion` 기능은 일절 건드리지 않는다. 두 기능은 감지 키워드
  (`TODO`/`Done` vs `체크리스트:`)가 달라 한 노트에서 공존 가능하다.
- `noteContentArchiver.ts` 무수정 — 체크리스트 항목은 평범한 `listItem`,
  마커는 평범한 텍스트.
- 모든 UI 문자열은 한국어 (CLAUDE.md 관례).
- `autoWeekday` 와 무간섭: autoWeekday 는 `N월` 헤더 아래에서만 동작하며
  `체크리스트:` 영역과 겹치지 않는다.
